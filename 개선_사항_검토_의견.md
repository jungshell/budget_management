# 📋 개선 사항 검토 의견

## 🎯 요청하신 개선 사항

1. **대시보드에 더 다양한 기준으로 분류, 분석한 시각화 자료 추가**
2. **예산 관련 중복 표시 제거** (대시보드와 예산관리 중복 → 예산관리에만)
3. **사이드바 2단계 뎁스 확장** (필요에 따라)

---

## 💡 검토 의견

### ✅ 1. 대시보드 시각화 다양화 - **매우 좋은 아이디어!**

#### 현재 대시보드 시각화
- ✅ 부서별 예산 분포 (막대 그래프)
- ✅ 재원별 구성 (파이 차트)
- ✅ 연도별 비교 (선 그래프)
- ✅ Treemap (부서별 예산)
- ✅ Heatmap (부서 x 재원 유형)
- ✅ 지도 (시군별 예산)

#### 추가하면 좋을 시각화

**📊 시간 기준 분석:**
- 📈 월별/분기별 예산 추이 (시계열 차트)
- 📅 예산 집행 일정 (Gantt 차트)
- 🔄 전년 동월 대비 비교

**🏢 조직 기준 분석:**
- 📋 부서별 예산 효율성 (스코어카드)
- 👥 담당자별 예산 관리 현황
- 🏛️ 상위/하위 부서 랭킹

**💰 재원 기준 분석:**
- 💵 재원 유형별 증감률 (증감률 차트)
- 📊 재원 조합 분석 (버블 차트)
- 💼 재원별 집행률 (게이지 차트)

**📈 트렌드 및 예측:**
- 📉 예산 추세 예측 (시계열 예측)
- 🎯 목표 대비 달성률 (레이더 차트)
- 📊 예산 분산도 (박스 플롯)

**🔍 상세 분석:**
- 📌 주요 사업별 상세 분석 (드릴다운)
- 🔗 사업 간 연관성 분석 (네트워크 그래프)
- 📊 예산 구성 비율 (산키 다이어그램)

#### 구현 제안
```
대시보드 구조:
├─ 📊 요약 카드 (현재 유지)
├─ 📈 주요 지표 (KPI)
├─ 📊 시각화 섹션 (탭 또는 아코디언)
│   ├─ 시간별 분석 탭
│   ├─ 조직별 분석 탭
│   ├─ 재원별 분석 탭
│   ├─ 트렌드 분석 탭
│   └─ 상세 분석 탭
├─ 🤖 AI 분석 섹션 (현재 유지)
└─ ⚠️ 알림 섹션 (현재 유지)
```

**장점:**
- ✅ 다양한 관점에서 데이터 분석 가능
- ✅ 의사결정에 도움되는 인사이트 제공
- ✅ 사용자가 원하는 분석 선택 가능

**주의사항:**
- ⚠️ 너무 많은 차트는 오히려 혼란스러울 수 있음
- ⚠️ 탭 또는 아코디언으로 정리 필요
- ⚠️ 성능 고려 (가상 스크롤, 지연 로딩)

---

### ✅ 2. 예산 표 중복 제거 - **완전 동의!**

#### 현재 상황
- 대시보드 하단에 `BudgetTable` 컴포넌트 표시
- 예산 관리 페이지에도 동일한 테이블 표시
- **중복으로 인한 혼란 및 불필요한 데이터 로딩**

#### 개선 방안

**옵션 1: 대시보드에서 완전 제거 (추천)**
```
✅ 장점:
- 대시보드는 요약과 시각화에 집중
- 예산 관리 페이지가 상세 데이터의 단일 소스
- 페이지 역할이 명확해짐
- 성능 개선 (불필요한 데이터 로딩 제거)

❌ 단점:
- 대시보드에서 바로 예산 수정 불가 (하지만 예산 관리 페이지로 이동하면 됨)
```

**옵션 2: 대시보드에 요약 테이블만 표시**
```
✅ 장점:
- 주요 항목만 빠르게 확인 가능
- 상세는 예산 관리 페이지로 이동

❌ 단점:
- 여전히 중복 가능성
- 어떤 항목을 보여줄지 결정 필요
```

**옵션 3: 링크로 대체**
```
✅ 장점:
- 중복 제거
- 명확한 네비게이션

구현:
"📊 전체 예산 보기" 버튼 → 예산 관리 페이지로 이동
```

#### 최종 추천: **옵션 1 (완전 제거)**
- 대시보드의 역할: **요약, 시각화, 인사이트**
- 예산 관리의 역할: **상세 데이터 관리**
- 역할 분리가 명확해짐

---

### ✅ 3. 사이드바 2단계 뎁스 확장 - **유연성 향상!**

#### 현재 상황
- 1단계 뎁스: 모든 메뉴가 평면적으로 표시
- 메뉴 항목이 많아지면 스크롤 필요

#### 개선 방안

**구조 예시:**
```
📊 예산 관리
  ├─ 📋 예산 목록
  ├─ ➕ 예산 추가
  └─ 📊 예산 분석

📁 데이터 관리
  ├─ 📤 파일 업로드
  ├─ 📥 데이터 내보내기
  └─ 💾 백업/복원

📄 리포트
  ├─ 📝 리포트 생성
  ├─ ⏰ 스케줄링
  └─ 🔗 공유 관리

⚙️ 설정
  ├─ 👤 사용자 관리
  ├─ 🤖 AI 설정
  └─ 🔗 연동 설정
```

#### 구현 방식

**옵션 1: 아코디언 방식 (추천)**
```
✅ 장점:
- 공간 효율적
- 확장/축소 가능
- 직관적

구현:
- Collapse 컴포넌트 사용
- 아이콘으로 확장/축소 표시
```

**옵션 2: 호버 방식**
```
✅ 장점:
- 클릭 불필요
- 빠른 접근

❌ 단점:
- 모바일에서 어려움
- 실수로 열림 가능
```

**옵션 3: 서브메뉴 드롭다운**
```
✅ 장점:
- 전통적인 방식
- 사용자 친숙

구현:
- Menu 컴포넌트 사용
- 클릭 시 서브메뉴 표시
```

#### 최종 추천: **옵션 1 (아코디언 방식)**
- Material-UI의 `Collapse` 컴포넌트 활용
- 사이드바가 닫혀있을 때는 1단계만 표시
- 사이드바가 열려있을 때는 2단계 확장 가능

#### 주의사항
- ⚠️ 너무 깊은 뎁스는 피하기 (최대 2단계 권장)
- ⚠️ 자주 사용하는 기능은 1단계에 유지
- ⚠️ 모바일에서는 1단계만 표시하는 것도 고려

## 🎯 종합 의견

### ✅ 모든 개선 사항이 타당합니다!

1. **대시보드 시각화 다양화**
   - 👍 매우 좋은 아이디어
   - 다양한 관점의 분석 제공
   - 탭/아코디언으로 정리 필요

2. **예산 표 중복 제거**
   - 👍 완전 동의
   - 역할 분리 명확화
   - 성능 개선

3. **사이드바 2단계 뎁스**
   - 👍 유연성 향상
   - 아코디언 방식 추천
   - 확장 가능한 구조

### 📊 우선순위 제안

**1단계 (즉시 적용):**
- 예산 표 중복 제거 (가장 간단하고 효과적)

**2단계 (단기):**
- 사이드바 2단계 뎁스 구현
- 대시보드 시각화 추가 (핵심 몇 개만)

**3단계 (중기):**
- 대시보드 시각화 확대
- 사용자 피드백 반영

### 💡 추가 제안

**대시보드 개인화:**
- 사용자가 원하는 차트만 선택하여 표시
- 위젯 드래그 앤 드롭으로 재배치
- 대시보드 템플릿 저장

**사이드바 스마트 메뉴:**
- 자주 사용하는 메뉴는 1단계에 유지
- 사용 빈도에 따라 자동 정렬
- 즐겨찾기 기능
