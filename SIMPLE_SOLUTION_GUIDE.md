# 간단한 해결 가이드 - CORS 문제 해결

## 현재 문제
- Google Apps Script와 프론트엔드 연동 시 CORS 오류 발생
- 복잡한 설정과 코드 수정으로 인한 혼란

## 해결 방법 선택

### 방법 1: Google Apps Script 배포 재설정 (가장 간단)

**단계별 체크리스트:**

1. ✅ Google Apps Script 편집기 열기
   - https://script.google.com 접속
   - 프로젝트 선택

2. ✅ 코드 확인
   - `GOOGLE_APPS_SCRIPT_FULL_CODE.js` 파일의 전체 내용 복사
   - Google Apps Script 편집기에 붙여넣기
   - 저장 (Ctrl+S)

3. ✅ 배포 삭제 후 재생성
   - `배포` → `배포 관리` 클릭
   - 기존 배포 삭제 (휴지통 아이콘)
   - `배포` → `새 배포` 클릭
   - 유형: `웹 앱` 선택
   - 설정:
     - 설명: "예산관리시스템"
     - 실행 대상: `나`
     - 액세스 권한: `모든 사용자` ⚠️ **반드시!**
   - `배포` 클릭
   - URL 복사

4. ✅ 프론트엔드 설정
   - 설정 페이지로 이동
   - "Google Apps Script Web App URL"에 복사한 URL 입력
   - 저장

5. ✅ 테스트
   - "표준 형식으로 Google Sheets 생성" 버튼 클릭
   - 새 탭에서 Google Sheets가 열리는지 확인

---

### 방법 2: CORS 우회 (프록시 서버 사용)

만약 방법 1이 작동하지 않으면, 백엔드 서버를 통해 프록시를 사용할 수 있습니다.

**장점:**
- CORS 문제 완전 해결
- 더 안정적인 연동

**단점:**
- 백엔드 코드 수정 필요

---

### 방법 3: Google Sheets API 직접 사용 (가장 안정적)

Google Apps Script 대신 Google Sheets API를 직접 사용하는 방법입니다.

**장점:**
- CORS 문제 없음
- 더 빠르고 안정적
- 공식 API 사용

**단점:**
- OAuth 인증 설정 필요
- API 키 관리 필요

---

## 추천: 방법 1부터 시도

가장 간단하고 빠른 방법은 **방법 1**입니다.

### 핵심 포인트
1. **기존 배포를 완전히 삭제**하고 새로 만들기
2. **"액세스 권한: 모든 사용자"** 반드시 설정
3. **새 URL을 설정 페이지에 입력**

### 문제가 계속되면
방법 2 또는 방법 3으로 전환하는 것을 고려하세요.
