# 오류 디버깅 가이드

## Cloud 로그가 활성화되지 않은 경우

Cloud 로그를 확인할 수 없어도, 오류 메시지를 프론트엔드에서 확인할 수 있도록 수정했습니다.

## 확인 방법

### 1단계: 코드 업데이트 및 재배포

1. `GOOGLE_APPS_SCRIPT_FULL_CODE.js` 파일의 전체 내용을 Google Apps Script에 복사
2. 저장 (Ctrl+S)
3. **배포** → **배포 관리** → 기존 배포 편집
4. **버전: "새 버전"** 선택
5. **배포** 클릭

### 2단계: 프론트엔드에서 테스트

1. 파일 업로드 페이지로 이동
2. **브라우저 개발자 도구 열기** (F12)
3. **Console 탭** 열기
4. "표준 형식으로 Google Sheets 생성" 버튼 클릭
5. **Console에 나타나는 오류 메시지 확인**

### 3단계: 오류 메시지 확인

Console에 다음과 같은 정보가 표시됩니다:

```
Google Apps Script 응답: {...}
동기화 실패: ...
오류 메시지: ...
스택: ...
```

이 정보를 복사해서 알려주시면 정확한 원인을 파악할 수 있습니다.

## 일반적인 오류와 해결 방법

### 오류 1: "POST 데이터가 없습니다"
**원인**: 프론트엔드에서 데이터가 제대로 전송되지 않음
**해결**: Network 탭에서 요청의 body 확인

### 오류 2: "JSON 파싱 오류"
**원인**: 전송된 데이터 형식 문제
**해결**: 프론트엔드 코드 확인

### 오류 3: "권한 오류" 또는 "Access denied"
**원인**: Google Sheets/Drive 접근 권한 없음
**해결**: 권한 재승인 필요

### 오류 4: "syncToSheets 오류"
**원인**: Google Sheets 생성/수정 중 오류
**해결**: 오류 메시지의 상세 내용 확인
