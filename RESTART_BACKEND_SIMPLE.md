# 백엔드 재시작 - 간단한 방법

## 현재 상황
- 백엔드 서버가 실행 중이지만 새 프록시 엔드포인트를 사용하려면 재시작 필요

## 재시작 방법

### 터미널에서 실행 (가장 확실한 방법)

1. **새 터미널 창 열기** (또는 기존 터미널 사용)

2. **다음 명령어 실행:**
   ```bash
   cd /Volumes/Samsung\ USB/budget_management_anti/functions
   python3 main.py
   ```

3. **정상적으로 시작되면 다음과 같은 메시지가 보입니다:**
   ```
   ✅ Firebase 초기화 성공
   ✅ Firestore 초기화 성공
    * Running on http://0.0.0.0:5001
   ```

4. **이 터미널 창은 열어둔 채로 유지** (서버가 계속 실행됨)

---

## 기존 서버 종료 방법

만약 다른 터미널에서 백엔드가 실행 중이라면:

1. **실행 중인 터미널 찾기**
2. **`Ctrl+C` 눌러서 종료**
3. **위의 재시작 방법으로 다시 시작**

---

## 재시작 후 테스트

1. **파일 업로드 페이지로 이동**
2. **"표준 형식으로 Google Sheets 생성" 버튼 클릭**
3. **CORS 오류 없이 작동해야 합니다!**

---

## 문제 해결

### "포트 5001이 이미 사용 중" 오류
```bash
# 포트 사용 중인 프로세스 확인
lsof -i :5001

# 프로세스 종료 (PID는 위 명령어 결과에서 확인)
kill [PID]
```

### "모듈을 찾을 수 없습니다" 오류
```bash
cd /Volumes/Samsung\ USB/budget_management_anti/functions
pip3 install requests
```
