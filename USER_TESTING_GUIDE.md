# 실제 사용자 테스트 가이드

## 개요
이 문서는 예산 관리 시스템의 실제 사용자 테스트를 위한 가이드입니다. 테스터가 시스템의 주요 기능을 검증하고 피드백을 제공할 수 있도록 구성되었습니다.

## 테스트 환경
- **브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
- **디바이스**: 데스크톱, 태블릿, 모바일
- **인터넷 연결**: 안정적인 인터넷 연결 필요

## 테스트 시나리오

### 1. 초기 접속 및 로그인
**목표**: 사용자가 시스템에 접속하고 로그인할 수 있는지 확인

**단계**:
1. 브라우저에서 애플리케이션 URL 접속
2. 로그인 화면 확인
3. Google 계정으로 로그인 (또는 이메일/비밀번호)
4. 로그인 성공 후 대시보드 표시 확인

**확인 사항**:
- [ ] 로그인 화면이 정상적으로 표시됨
- [ ] 로그인 버튼이 정상 작동함
- [ ] 로그인 후 대시보드로 이동함
- [ ] 에러 메시지가 명확함 (로그인 실패 시)

**예상 소요 시간**: 2-3분

---

### 2. 대시보드 탐색
**목표**: 대시보드의 주요 정보와 차트를 확인

**단계**:
1. 대시보드 페이지 로드 확인
2. 예산 요약 카드 확인 (총 예산, 국비, 도비, 시군비, 자체)
3. 차트 확인 (사업별 출연금액, 재원별 구성 비율, 연도별 예산 비교)
4. 최근 활동 내역 확인

**확인 사항**:
- [ ] 모든 예산 요약 카드가 정상적으로 표시됨
- [ ] 차트가 정상적으로 렌더링됨
- [ ] 차트 데이터가 정확함
- [ ] 숫자에 천 단위 구분 쉼표가 적용됨
- [ ] 로딩 상태가 적절히 표시됨

**예상 소요 시간**: 3-5분

---

### 3. 사이드바 기능
**목표**: 사이드바의 연도/회차 선택 및 메뉴 네비게이션 확인

**단계**:
1. 사이드바 열림/닫힘 토글
2. 사이드바 열린 상태에서 연도 선택
3. 사이드바 열린 상태에서 회차 선택
4. 사이드바 닫힌 상태에서 연도/회차 표시 확인
5. 메뉴 항목 클릭하여 페이지 이동

**확인 사항**:
- [ ] 사이드바 토글이 정상 작동함
- [ ] 연도 선택이 정상 작동함
- [ ] 회차 선택이 정상 작동함
- [ ] 사이드바 닫힌 상태에서 연도/회차가 가독성 있게 표시됨
- [ ] 메뉴 클릭 시 해당 페이지로 이동함

**예상 소요 시간**: 3-4분

---

### 4. 예산 관리 페이지
**목표**: 예산 항목 조회, 검색, 필터링 기능 확인

**단계**:
1. 예산 관리 페이지로 이동
2. 예산 목록 확인
3. 검색 기능 테스트 (사업명, 부서명)
4. 부서 필터 테스트
5. 페이징 기능 확인

**확인 사항**:
- [ ] 예산 목록이 정상적으로 표시됨
- [ ] 검색 기능이 정상 작동함
- [ ] 필터 기능이 정상 작동함
- [ ] 페이징이 정상 작동함
- [ ] 테이블 정렬이 정상 작동함

**예상 소요 시간**: 5-7분

---

### 5. 예산 항목 추가/수정/삭제
**목표**: 예산 항목의 CRUD 기능 확인

**단계**:
1. 예산 항목 추가 버튼 클릭
2. 필수 필드 입력 (사업명, 부서, 예산 금액 등)
3. 저장 후 목록에 반영 확인
4. 기존 항목 수정
5. 항목 삭제
6. 일괄 선택 및 일괄 삭제

**확인 사항**:
- [ ] 추가 다이얼로그가 정상적으로 열림
- [ ] 필수 필드 검증이 정상 작동함
- [ ] 저장 후 목록에 반영됨
- [ ] 수정 기능이 정상 작동함
- [ ] 삭제 기능이 정상 작동함
- [ ] 일괄 삭제가 정상 작동함
- [ ] 권한이 없는 사용자는 수정/삭제 불가

**예상 소요 시간**: 7-10분

---

### 6. 데이터 내보내기
**목표**: 엑셀/CSV 내보내기 기능 확인

**단계**:
1. 예산 관리 페이지에서 내보내기 버튼 클릭
2. 엑셀 형식 선택
3. 다운로드된 파일 확인
4. CSV 형식 선택
5. 다운로드된 파일 확인

**확인 사항**:
- [ ] 엑셀 파일이 정상적으로 다운로드됨
- [ ] CSV 파일이 정상적으로 다운로드됨
- [ ] 파일 내용이 정확함
- [ ] 천 단위 구분 쉼표가 적용됨
- [ ] 파일명이 적절함

**예상 소요 시간**: 3-5분

---

### 7. 파일 업로드
**목표**: 엑셀 파일 업로드 및 파싱 기능 확인

**단계**:
1. 업로드 페이지로 이동
2. 엑셀 파일 선택
3. 파일 업로드
4. 미리보기 확인
5. 데이터 저장

**확인 사항**:
- [ ] 파일 선택이 정상 작동함
- [ ] 파일 업로드가 정상 작동함
- [ ] 미리보기가 정상적으로 표시됨
- [ ] 데이터 파싱이 정확함
- [ ] 에러 메시지가 명확함 (잘못된 형식 시)

**예상 소요 시간**: 5-7분

---

### 8. AI 분석 기능
**목표**: AI 기반 예산 분석 기능 확인

**단계**:
1. 분석 페이지로 이동
2. 예시 질문 클릭 또는 직접 질문 입력
3. 분석 결과 확인
4. 질문 히스토리 확인

**확인 사항**:
- [ ] 질문 입력이 정상 작동함
- [ ] 분석 결과가 정상적으로 표시됨
- [ ] 로딩 상태가 적절히 표시됨
- [ ] 질문 히스토리가 저장됨
- [ ] 에러 메시지가 명확함

**예상 소요 시간**: 5-7분

---

### 9. 설정 페이지
**목표**: 사용자 설정 기능 확인

**단계**:
1. 설정 페이지로 이동
2. 설정 옵션 확인
3. 설정 변경
4. 변경사항 저장

**확인 사항**:
- [ ] 설정 옵션이 정상적으로 표시됨
- [ ] 설정 변경이 정상 작동함
- [ ] 변경사항이 저장됨

**예상 소요 시간**: 3-5분

---

### 10. 키보드 단축키
**목표**: 키보드 단축키 기능 확인

**단계**:
1. 예산 관리 페이지에서 Ctrl+N (추가)
2. Ctrl+S (저장)
3. Esc (닫기)
4. 기타 단축키 테스트

**확인 사항**:
- [ ] Ctrl+N이 정상 작동함
- [ ] Ctrl+S가 정상 작동함
- [ ] Esc가 정상 작동함
- [ ] 단축키 충돌이 없음

**예상 소요 시간**: 2-3분

---

### 11. 반응형 디자인
**목표**: 다양한 화면 크기에서의 UI 확인

**단계**:
1. 브라우저 창 크기 조절 (데스크톱, 태블릿, 모바일)
2. 각 페이지에서 레이아웃 확인
3. 모바일에서 터치 제스처 확인

**확인 사항**:
- [ ] 데스크톱에서 레이아웃이 정상적임
- [ ] 태블릿에서 레이아웃이 정상적임
- [ ] 모바일에서 레이아웃이 정상적임
- [ ] 텍스트가 가독 가능함
- [ ] 버튼이 클릭 가능한 크기임

**예상 소요 시간**: 5-7분

---

### 12. 성능 테스트
**목표**: 페이지 로딩 속도 및 반응성 확인

**단계**:
1. 각 페이지 로딩 시간 측정
2. 대용량 데이터 로딩 테스트
3. 차트 렌더링 속도 확인
4. 검색/필터 반응 속도 확인

**확인 사항**:
- [ ] 페이지 로딩이 빠름 (< 3초)
- [ ] 대용량 데이터도 정상 처리됨
- [ ] 차트 렌더링이 빠름
- [ ] 검색/필터 반응이 빠름
- [ ] 로딩 인디케이터가 적절함

**예상 소요 시간**: 5-7분

---

### 13. 에러 처리
**목표**: 에러 상황에서의 사용자 경험 확인

**단계**:
1. 네트워크 연결 끊기
2. 잘못된 데이터 입력
3. 권한 없는 작업 시도
4. 에러 메시지 확인

**확인 사항**:
- [ ] 네트워크 오류 시 적절한 메시지 표시
- [ ] 잘못된 입력 시 명확한 에러 메시지
- [ ] 권한 오류 시 적절한 안내
- [ ] 에러 복구가 가능함

**예상 소요 시간**: 5-7분
